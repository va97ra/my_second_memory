# Архитектура

Проект разбит по фичам и слоям, чтобы новые типы памяти можно было добавлять без переписывания ленты и экранов.

## Слои

- `core`: тема, маршруты, локализация, общая инфраструктура.
- `shared`: переиспользуемые UI-обертки.
- `features/*/domain`: модели и правила.
- `features/*/data`: работа с устройством, файлами, базой, сервисами.
- `features/*/state`: Riverpod-контроллеры.
- `features/*/ui`: экраны и маленькие виджеты.
- `data/database`: активная Drift/SQLite-схема и миграции.

## Текущее хранилище

На Android и desktop активен `SqliteMemoryRepository`. Он изменяет отдельные строки через `upsert` и `delete`; полная замена `replaceAll` используется только при восстановлении резервной копии. Один экземпляр `AppDatabase` живёт всю сессию Riverpod-контейнера и закрывается вместе с ним.

На Web временно используется `LocalMemoryRepository` через `SharedPreferences`. UI зависит только от `MemoryRepository` и не знает, какое хранилище выбрано платформой.

При включённом PIN записи, аккаунты и смены хранятся отдельными AES-GCM-зашифрованными строками Drift. Фото и голос также шифруются; изображения расшифровываются в память, а голос - во временный файл только на время воспроизведения. Миграция сначала создаёт и проверяет новые копии и только затем удаляет исходные данные.

Новый резервный формат создаёт AES-256 ZIP потоково: `manifest.json` и медиа лежат отдельными файлами без Base64. Импорт старых ZIP и JSON остаётся поддержан.

## Состояние и навигация

Четыре основные вкладки работают через `StatefulShellRoute.indexedStack`, поэтому их прокрутка и локальное состояние не уничтожаются при переключении. Производные Riverpod-провайдеры отдают запись по ID, записи дня, архив, группы ленты и записи видимой сетки календаря.

Автосохранение редактора и запись в репозиторий выполняются последовательно: старая операция не может перезаписать более новую правку. Ожидающее изменение сбрасывается при возврате назад и сворачивании приложения.

## Правило дат

Каждая запись имеет обязательное поле `memoryDate`. Оно отвечает за появление записи в ленте и календаре. Поле `remindAt` предназначено для уведомлений и не заменяет дату записи.

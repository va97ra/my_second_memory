# Архитектура

Проект разбит по фичам и слоям, чтобы новые типы памяти можно было добавлять без переписывания ленты и экранов.

## Слои

- `core`: тема, маршруты, локализация, общая инфраструктура.
- `shared`: переиспользуемые UI-обертки.
- `features/*/domain`: модели и правила.
- `features/*/data`: работа с устройством, файлами, базой, сервисами.
- `features/*/state`: Riverpod-контроллеры.
- `features/*/ui`: экраны и маленькие виджеты.
- `data/database`: Drift-таблицы для будущей SQLite-реализации.

## Текущее хранилище

Активный репозиторий `LocalMemoryRepository` сохраняет записи локально через `SharedPreferences`, чтобы MVP можно было поднять без генерации Drift-кода. UI зависит только от интерфейса `MemoryRepository`, поэтому переход на Drift/SQLite должен затронуть только data-слой.

Drift/sqlite3 подключены как активные зависимости. Пока UI работает через `MemoryRepository`, поэтому переход с `SharedPreferences` на SQLite должен быть локальной заменой data-слоя.

## Правило дат

Каждая запись имеет обязательное поле `memoryDate`. Оно отвечает за появление записи в ленте и календаре. Поле `remindAt` предназначено для уведомлений и не заменяет дату записи.
